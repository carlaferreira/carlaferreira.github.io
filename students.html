Open MSc thesis proposals (2026/27)

  • Automated Dependency and Contract Inference for RESTful APIs
    ATLAS ref. MEI2627/069
    Reduce the manual effort of formal-methods-based API testing by automatically inferring operation dependencies and executable Glacier contracts from OpenAPI specifications, including an LLM-assisted module for extracting semantics from natural-language descriptions. Builds on the IcePick framework; the student joins the IcePick team at NOVA LINCS.
    Keywords: LLM-assisted software engineering · RESTful APIs · formal-methods testing
  • Automated Inference of Likely Invariants and TLA+ Specifications for RESTful APIs
    ATLAS ref. MEI2627/070
    Automatically infer likely behavioural invariants for REST APIs and generate TLA+ specifications from OpenAPI descriptions enriched with Glacier contracts, with abstraction heuristics to keep models tractable for model checking. Builds on IcePick; empirical evaluation on real-world APIs.
    Keywords: API testing · TLA+ · invariant detection · black-box testing
  • Runtime Checking of Privacy and Security Policies
    ATLAS ref. MEI2627/068
    Runtime verification techniques for enforcing privacy and security policies by monitoring information flows during execution: a policy language for context-sensitive requirements, a prototype runtime monitor, and mechanisms to report or block violations.
    Keywords: information-flow control · runtime verification · language-based security
  • Language-Vision Models for Automated Ontology Generation from Unstructured Text and Diagrams: Where Software Verification meets AI
    Co-supervision with Cláudia Soares, in collaboration with the Portuguese Oncology Institute of Lisbon (IPO-L)
    How can language-vision models be combined with formal verification to generate trustworthy medical ontologies from clinical guidelines expressed as text and flowcharts? LLMs and VLMs propose candidate ontologies, while model-checking-inspired techniques validate them — representing clinical decision pathways as state-transition systems, treating guideline-derived constraints as executable semantic contracts, and systematically exploring reachable clinical states to reveal omissions, contradictions, or unsafe recommendations. Continues our ICST 2026 work on ACTORCHESTRA, a runtime verification framework for actor-based systems.
    Keywords: LLMs/VLMs · ontologies · model checking · healthcare
